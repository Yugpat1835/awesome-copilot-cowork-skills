# Voice Profile Template

BUILD mode of the write-like-me skill fills this template and saves the result as voice-profile.md. Keep the section headings exactly as written here so DRAFT mode can find each one. Replace every placeholder with evidence from the user's own sent mail and speech turns. If a section has no evidence, write "insufficient data" rather than guessing.

## Metadata

- Built on: YYYY-MM-DD
- Sent messages analysed: N
- Transcripts analysed: N
- Period covered: YYYY-MM-DD to YYYY-MM-DD
- Dominant language: x (note any secondary language and rough share)
- Confidence: High (30 or more messages) or Low
- Notes: anything that limits the profile (narrow topic mix, one audience only)

## Greetings

How the user opens messages, by audience. Record exact strings and rough frequency.

- Internal, familiar colleagues: exact greeting, how often
- Internal, senior or formal: exact greeting, how often
- External contacts: exact greeting, how often
- No greeting, straight into the point: how often, and in what situations

## Sign-offs

Exact closing lines, including punctuation and name form (full name, first name, initial).

- Usual sign-off: exact string
- Formal sign-off: exact string
- When the sign-off is skipped: situations

## Sentence and paragraph length

- Average sentence length: N words (typical range N to N)
- Typical paragraph: N sentences
- Typical email length: N words
- Behaviour in longer documents, if evidence exists

## Formality and tone

- Default register: one plain sentence describing it
- When the user shifts more formal, and how that shows
- When the user shifts more casual, and how that shows
- Directness: hedged or blunt, with one short excerpt as evidence

## Signature vocabulary

Words and phrases the user reaches for repeatedly. Up to 15 items, each with a note on where it appears (openers, asks, closings).

## Never uses

Words, phrases and patterns absent from the evidence or visibly avoided. DRAFT mode must not use anything listed here. Check for: corporate buzzwords, emoji, exclamation marks, all-caps emphasis, greetings the user never picks, sign-offs the user never picks.

## Structure habits

- Bullets versus paragraphs: which the user prefers and when
- How messages open: context first, ask first, or answer first
- How actions are handed off: named owner, deadline phrasing, follow-up phrasing
- Subject line style for emails: length, capitalisation, prefixes

## Punctuation and formatting

- Dash, semicolon and parenthesis habits
- Bold, italics and numbered list habits
- Link habits: bare URL, named link, or "see attached"

## Spoken versus written differences

Fill only if transcripts were analysed. List speech tics that must not leak into writing, and phrases that work in both speech and writing.

## Dos and donts

Distil everything above into rules a drafter can apply directly.

- Do: 3 to 6 rules
- Dont: 3 to 6 rules

## Example excerpts

Up to three excerpts per register (familiar, formal, external), each under 25 words, user's own words only, no third-party names.
