# actions.md template

Produce the actions file with exactly this structure. Replace angle-bracket placeholders. Keep section order. Leave a section's body as "None found." rather than deleting the section.

```markdown
# DRAFT: Action items, <Meeting name>, <YYYY-MM-DD>

> Generated by the transcript-to-actions Cowork skill on <run date YYYY-MM-DD>. DRAFT until reviewed by a human.

Source transcript: <file path, meeting name, or "pasted text">
Attendees (as listed): <names, or "Not listed">

## Decisions

1. <Decision in one sentence.> Source: "<verbatim fragment>"

## Action items

| # | Owner | Action | Due | Source quote | Task write status |
|---|---|---|---|---|---|
| 1 | <Name or Unassigned> | <Verb-led action> | <YYYY-MM-DD or No date given> | "<fragment>" | <To Do: confirmed / Planner: confirmed / file only (write failed or unconfirmed) / not attempted (file only chosen)> |

## Open questions

1. <Question.> Raised by <name or "unclear">. Source: "<verbatim fragment>"

## Write log

- Actions file: saved to <full OneDrive path>
- Task destination chosen: <To Do / Planner plan "<name>" / file only>
- Task writes confirmed: <n> of <total>
- Email drafts created: <n> (owners: <names>); drafts without a resolvable recipient: <names or "none">
```

Rules:

- The title line and the blockquote must both carry the word DRAFT.
- The Task write status column starts as "pending" when the file is first written, then is updated in place after the task-write step. This in-run update is the only permitted edit to an existing file.
- Do not add sections beyond these five.
